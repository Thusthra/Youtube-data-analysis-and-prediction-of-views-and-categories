About This Project

YouTube Data Analysis and Prediction System

Project Overview

This project focuses on analyzing YouTube video data to extract meaningful insights and build predictive models for estimating video performance and classification.

Using data collected through the YouTube Data API or publicly available datasets, the system performs exploratory data analysis to understand trends in views, categories, user engagement, and publishing patterns.

Machine learning techniques are applied to predict the number of views a video may receive and to classify videos into appropriate categories based on textual features such as titles, descriptions, and tags.

Key Features

Data Collection

Fetch data using YouTube Data API or load from CSV/JSON datasets. Extract video titles, views, likes, comments, categories, and tags.

Data Preprocessing

Handle missing values, remove duplicates, convert data types, clean text data, and create new features for analysis.

Exploratory Analysis

Visualize trends using interactive charts. Identify popular categories, peak upload times, and correlations between metrics.

View Prediction

Train regression models (Linear Regression, Random Forest, Gradient Boosting) to predict expected view counts for videos.

Category Prediction

Apply NLP using TF-IDF and train classifiers (Logistic Regression, Naive Bayes, SVM) to predict video categories.

Web Application

Interactive Flask web application with user-friendly interface for predictions, visualizations, and video comparisons.

How It Works

The system can fetch data from YouTube using the YouTube Data API or load pre-existing datasets in CSV/JSON format. It extracts key information including video titles, descriptions, tags, view counts, likes, comments, and categories.

Raw data is cleaned and preprocessed. This includes handling missing values, removing duplicates, converting data types, cleaning text data (removing URLs, special characters), and creating new features like title length, tag count, and upload time features.

Interactive visualizations are generated to understand data patterns. This includes distribution plots, correlation heatmaps, time series analysis, and category-wise statistics. Insights help identify trends and relationships in the data.

Multiple regression models are trained to predict view counts. Features include title length, tag count, description length, upload time, and engagement ratios. Models are evaluated using RMSE, MAE, and R² scores, with the best model selected for predictions.

Text features (title, description, tags) are vectorized using TF-IDF and used to train classification models. Logistic Regression, Naive Bayes, SVM, and Random Forest classifiers are evaluated, with the best model selected based on accuracy and F1-score.

A Flask web application provides an interactive interface for users. Features include video prediction forms, interactive dashboards, video comparison tools, and detailed analysis pages. The application uses Bootstrap for responsive design and Chart.js for client-side visualizations.
